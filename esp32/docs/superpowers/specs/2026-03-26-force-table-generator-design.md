# Force Table Generator

## Overview

A Python script that computes precomputed lateral force lookup tables for each of the 5 coil layer heights using Biot-Savart magnetic field simulation. Outputs a C++ header file with `constexpr` arrays that the ESP32 physics simulation uses for force lookup at runtime.

## Script Location

`simulations/generate_force_table.py` — lives alongside existing simulation code, imports from `coil_sim.py`.

## Configurable Parameters

```python
# Coil geometry
COIL_WIDTH_MM = 37.0        # square coil outer dimension
TRACE_WIDTH_MM = 0.4
TRACE_SPACING_MM = 0.1
INNER_DIAMETER_MM = 2.0     # inner opening half-side

# Magnet
MAGNET_DIAMETER_MM = 5.0
MAGNET_HEIGHT_MM = 2.0
MAGNET_BR = 1.3             # N42 remanence (T)

# Grid
GRID_RESOLUTION_MM = 1.0    # step size (configurable for finer resolution later)
GRID_EXTENT_MM = 19.0       # half pitch — max distance from coil center

# Layer heights (distance from magnet surface to coil plane)
LAYER_DISTANCES_MM = [0.5, 0.75, 1.0, 1.25, 1.5]

# Current normalization
CURRENT_A = 1.0             # all forces computed at 1A

# Output
OUTPUT_PATH = "../esp32/firmware/force_tables.h"
```

## Computation

1. Build a single square rectangular spiral coil at origin using `build_rectangular_spiral` from `coil_sim.py`
2. For each of 5 layer heights:
   a. Compute 3D B-field on a grid around the coil using Biot-Savart (reuses `_biot_savart_segment`)
   b. Compute dBz/dx and dBz/dy field gradients via `numpy.gradient`
   c. Model the magnet as a dipole grid (reuses existing magnet model from `coil_sim.py`)
   d. Compute Fx and Fy at each point in the full grid (-19 to +19mm in both axes)
3. Write output header file

Uses field caching (pickle files) like existing simulation scripts to speed up repeated runs.

## Full Matrix Storage

The table covers the full field, not just one quadrant. At 1mm resolution:
- Grid size: 39x39 (from -19mm to +19mm inclusive)
- Per layer: 39 × 39 × 2 floats (Fx, Fy) = 12,168 bytes
- 5 layers: ~60KB total
- Fits easily in ESP32's 4MB flash

No mirroring logic needed at runtime — firmware just does bounds check and bilinear interpolation.

## Output Header Format

`esp32/firmware/force_tables.h`:

```cpp
// Auto-generated by simulations/generate_force_table.py
// DO NOT EDIT — regenerate with: cd simulations && python generate_force_table.py
//
// Coil: 37.0mm square, 35 turns, 0.4mm trace, 0.1mm spacing, 2.0mm inner dia
// Magnet: 5.0mm dia, 2.0mm height, N42 (1.3T remanence)
// Current: 1.0A (multiply by actual current at runtime)
// Resolution: 1.0mm
// Grid: -19mm to +19mm (39x39 per layer)
// Layer distances from magnet: [0.5, 0.75, 1.0, 1.25, 1.5] mm
//
// force_table_fx[layer][y_idx][x_idx] — lateral force in x (mN at 1A)
// force_table_fy[layer][y_idx][x_idx] — lateral force in y (mN at 1A)
// Index 0 = -19mm, index 19 = 0mm (center), index 38 = +19mm

#pragma once
#include <cstdint>

constexpr int FORCE_TABLE_SIZE = 39;
constexpr float FORCE_TABLE_RES_MM = 1.0f;
constexpr float FORCE_TABLE_EXTENT_MM = 19.0f;
constexpr int FORCE_TABLE_CENTER = 19;  // index of (0,0)
constexpr int FORCE_TABLE_NUM_LAYERS = 5;

constexpr float force_table_fx[5][39][39] = {
  // Layer 0 (0.5mm from magnet — closest, strongest)
  { { ... }, ... },
  // Layer 1 (0.75mm)
  { { ... }, ... },
  // Layer 2 (1.0mm)
  { { ... }, ... },
  // Layer 3 (1.25mm)
  { { ... }, ... },
  // Layer 4 (1.5mm — furthest, weakest)
  { { ... }, ... },
};

constexpr float force_table_fy[5][39][39] = {
  // same structure as fx
};
```

## Layer-to-Coil Mapping

Each coil position in the L-shape pattern maps to a layer:

| L-shape position | Local bit | Layer index | Distance to magnet |
|-----------------|-----------|-------------|-------------------|
| (0,0) | 2 | 0 | 0.5mm |
| (1,0) | 1 | 1 | 0.75mm |
| (2,0) | 0 | 2 | 1.0mm |
| (0,1) | 3 | 3 | 1.25mm |
| (0,2) | 4 | 4 | 1.5mm |

Firmware mapping function:
```cpp
static int bitToLayer(uint8_t local_bit) {
  constexpr int map[] = {2, 1, 0, 3, 4};
  return (local_bit < 5) ? map[local_bit] : 0;
}
```

## Runtime Force Lookup (firmware side, for reference)

Given piece position (px, py) in mm and coil center (cx, cy) in mm:

```cpp
float dx = px - cx;  // offset from coil center
float dy = py - cy;

// Clamp to table bounds
float fx_idx = (dx + FORCE_TABLE_EXTENT_MM) / FORCE_TABLE_RES_MM;
float fy_idx = (dy + FORCE_TABLE_EXTENT_MM) / FORCE_TABLE_RES_MM;
```

## Dependencies

- Imports `build_rectangular_spiral`, `_biot_savart_segment` from `coil_sim.py`
- Reuses magnet dipole model (magnetization grid, force from gradient)
- Uses numpy, scipy for field computation and interpolation
- Uses pickle for field caching

## What This Does NOT Cover

- Firmware physics rework (separate spec)
- PhysicsParams changes (separate spec)
- Frontend changes (none needed for this script)
